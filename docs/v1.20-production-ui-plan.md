# v1.20 生产级 UI + 内容运营后台清晰化 — 总体计划

> 上一版本：v1.19.5-stable  
> 目标版本：v1.20.5-stable  
> 日期：2026-05-16

---

## 目标

把现有项目从"功能原型"升级为"可正式运营的产品级网站"。

## 核心原则

1. **不允许自由发挥 UI** — 所有改动必须基于统一设计系统
2. **先建立设计系统，再改页面** — 不边想边做
3. **移动端优先** — 所有页面必须先适配手机
4. **截图验收** — 每次 UI 修改必须提供桌面端和移动端截图
5. **Build 通过 ≠ 验收通过** — 必须做视觉验收
6. **按阶段推进** — 不一口气改全站

---

## 当前 UI 问题诊断

### 1. 首页不够高级
- 首屏缺乏明确的 Hero 区域，用户打开不知道网站做什么
- 工具卡片风格不统一，间距不一致
- 缺少搜索入口和身份引导
- 底部广告位、内容推荐区域没有层次

### 2. 工具页粗糙
- 缺少工具说明区（这个工具做什么、怎么用）
- 输入区域和输出区域视觉上没有区分
- 没有示例/模板展示
- 缺少"相关工具"推荐
- 空状态不友好

### 3. 后台管理像半成品
- 后台首页模块卡片虽然清晰，但视觉上像后台框架
- 文章管理编辑器简陋
- 资源管理表格没有筛选可视化
- 缺少数据统计 Dashboard
- 导航层级不清晰

### 4. 单据模板差异太小
- 8 种唛头类型只有颜色差异，没有布局差异
- 6 种视觉风格主要是换色，缺乏商务感
- 缺少真实的商业单据布局（外箱唛头、集运标签、FBA 标签等应有不同结构）
- 打印效果未优化

### 5. 内容入口不够清楚
- 文章(/guides)、资源库(/resources)、工具(/tools)之间的关联弱
- 首页缺少内容推荐区域
- 导航栏不够直观
- 新用户不知道该从哪里开始

### 6. 移动端细节不足
- 部分页面横向溢出
- 按钮在小屏幕上不够大
- 表单在手机上布局拥挤
- 卡片间距在小屏幕上过大或过小

## 验收标准

- [ ] 首页看起来像正式产品，不像 demo
- [ ] 工具页结构清晰：说明 → 输入 → 输出 → 示例 → 相关工具
- [ ] 后台每个模块知道数据来自哪里、在哪里编辑、在哪里展示
- [ ] 广告管理支持本站自营广告（图片/文字/代码），不局限于第三方代码
- [ ] 单据模板必须有真实布局差异，不只是换颜色
- [ ] iPhone Safari 不横向溢出
- [ ] 所有核心页面截图通过视觉审查
- [ ] npm run build 0 errors
- [ ] PM2 online，所有核心页面 200

---

## 约束

- 不改数据库结构（除非必要）
- 不触碰 AI 额度/积分核心逻辑
- 不启用真实 AI Key（保持 Mock 模式）
- 不动会员/签到/兑换/单据数据逻辑
- 不改邮编库
