> 专题（Topics）与资源（Resources）功能详解
> 适用于：网站运营者 / 内容编辑
> 版本：v1.21.1 | 更新日期：2026-05-21

## 一、核心概念：专题 vs 资源 vs 文章

| 模块 | 数据库模型 | 定位 | 适合场景 |
|------|-----------|------|----------|
| **专题（Topic）** | `Topic` | 场景化内容合集 | 围绕一个主题打包多个 APP/工具评测，含 S/A/B/C/D 评级 |
| **资源（Resource）** | `Resource` | 第三方网站/工具导航链接 | 物流公司官网、支付平台、建站工具等外部资源收录 |
| **文章（Article）** | `Article` | 独立指南/教程/博客 | 集运攻略、清关指南、出海经验分享等长文 |

### 逻辑关系

```
专题 (Topic)
  ├── 专题基本信息（标题、封面、SEO）
  ├── 专题条目 (TopicItem) — 每个条目是一个 APP/工具的评测
  │     ├── 名称、别名、评级（S/A/B/C/D）
  │     ├── 国内类比、避坑提醒
  │     └── 官网链接、新手推荐标记
  ├── 专题章节 (TopicSection) — 补充说明文字块
  │     ├── 类型：intro / tips / warning / conclusion
  │     └── 内容：富文本段落
  └── YouTube 视频（可选）— 配套讲解视频

资源 (Resource) — 独立存在，不隶属于专题
  ├── 名称、URL、描述
  ├── 分类（life/logistics/business/templates）
  ├── 标签数组
  └── 来源类型（官方/第三方/内部）

文章 (Article) — 独立存在
  ├── 标题、Slug、正文内容
  ├── 封面图、分类
  └── SEO 标题与描述
```

## 二、后台：如何创建一个新专题

### 步骤 1：进入专题管理

1. 登录主站后，访问 **`https://jueshi.net/admin/topics`**
2. 点击「**新建专题**」按钮

### 步骤 2：填写基本信息（Basic 标签页）

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| **Slug** | ✅ | URL 标识，只能英文+数字+连字符 | `must-have-apps-2026` |
| **标题** | ✅ | 专题名称 | `出海之后必装 APP 评级推荐` |
| **副标题** | ❌ | 一句话补充说明 | `18 个 APP 从通讯到工作全覆盖` |
| **摘要** | ❌ | 专题简介，前端卡片展示 | `S/A/B/C/D 评级，每个都有国内类比和避坑提醒` |
| **状态** | ✅ | `draft`（草稿）/ `published`（已发布）/ `archived`（已归档） | 先选 `draft`，确认无误后改 `published` |
| **模板类型** | ✅ | `rating_list`（评级列表）— 当前唯一选项 | `rating_list` |
| **封面 Emoji** | ❌ | 专题图标 | `📱` |
| **YouTube 链接** | ❌ | 配套视频 URL，自动解析视频 ID 和封面 | `https://www.youtube.com/watch?v=xxx` |
| **SEO 标题** | ❌ | 搜索引擎展示标题 | `出海必装 APP 推荐 2026 - 海外百宝箱` |
| **SEO 描述** | ❌ | 搜索引擎展示描述 | `18 款海外生活必备 APP 评级推荐...` |

> **⚠️ Slug 注意事项：**
> - 创建后**不可修改**（数据库唯一约束）
> - 推荐使用英文短横线连接：`overseas-must-have-apps`
> - 前端 URL 为：`https://jueshi.net/topics/你的slug`

### 步骤 3：添加专题条目（Items 标签页）

每个专题条目对应一个 APP 或工具的评测：

| 字段 | 说明 |
|------|------|
| **名称** | APP 名称，如 `WhatsApp` |
| **别名** | 中文俗称，如 `色叉` |
| **评级** | `S` / `A` / `B` / `C` / `D` 五级 |
| **分类** | 如 `通讯` / `社交` / `工作` |
| **图标文字** | 缩写或 Emoji，如 `WA` |
| **国内类比** | 对应国内产品，如 `微信` |
| **适合谁** | 如 `所有人` / `留学生` |
| **新手建议** | 使用建议 |
| **风险提示** | 避坑提醒 |
| **官网 URL** | 官方下载链接 |
| **新手推荐** | 勾选后前端标记为「新手友好」 |
| **排序** | 数字越小越靠前 |

### 步骤 4：添加补充章节（Sections 标签页）

用于在专题中插入说明文字块：

| 类型 | 用途 |
|------|------|
| `intro` | 专题引言 |
| `tips` | 使用技巧/注意事项 |
| `warning` | 重要警告 |
| `conclusion` | 总结 |

### 步骤 5：发布专题

1. 确认所有内容填写完毕
2. 回到 **Basic** 标签页，将 **状态** 改为 `published`
3. 点击 **保存**
4. 访问 `https://jueshi.net/topics/你的slug` 查看前端效果

## 三、后台：如何发布一篇资源/文章

### 发布资源（Resource）

> 资源是外部网站/工具的导航链接，收录在「资源库」页面。

1. 访问 **`https://jueshi.net/admin/resources`**
2. 点击「**新增资源**」
3. 填写：
   - **名称**：网站/工具名称
   - **URL**：官方网址（必须是有效的 http/https 链接）
   - **描述**：一句话说明用途
   - **分类**：`life`（生活）/ `logistics`（物流）/ `business`（商业）/ `templates`（模板）
   - **标签**：多选标签，如 `集运`、`清关`
   - **来源类型**：`official`（官方）/ `third-party`（第三方）/ `internal`（内部）
   - **使用提示**（可选）：适用场景说明
   - **免责声明**（可选）
   - **排序**：数字越小越靠前
   - **是否启用**：勾选后前端可见
4. 保存后，资源将出现在 `https://jueshi.net/resources`

### 发布文章（Article）

> 文章是独立的指南/教程/博客长文。

1. 访问 **`https://jueshi.net/admin/cms`**（CMS 管理页）
2. 点击「**新建文章**」
3. 填写：
   - **标题**：文章标题
   - **Slug**：URL 标识（英文+连字符，如 `shipping-guide-2026`）
   - **正文**：Markdown 格式内容
   - **摘要**：一句话简介
   - **封面图 URL**：文章封面图片地址
   - **作者**：可选
   - **分类**：`跨境寄送` / `海外生活` / `出海经营`
   - **状态**：`draft`（草稿）/ `published`（已发布）
   - **SEO 标题/描述**：可选
4. 保存后，文章将出现在 `https://jueshi.net/guides/你的slug`

## 四、前端展示逻辑

### /topics — 专题列表页

```
┌─────────────────────────────────────┐
│  Hero 区：渐变色背景 + 标题 + 描述   │
├─────────────────────────────────────┤
│  已发布专题卡片网格（2 列响应式）      │
│  ┌───────────┐  ┌───────────┐       │
│  │ 📱 标题    │  │ 🛒 标题    │       │
│  │ 副标题      │  │ 副标题      │       │
│  │ 摘要...    │  │ 摘要...    │       │
│  │ [▶含视频]  │  │ [5个条目]  │       │
│  │ 👤 适合：xx │  │ 标签 标签  │       │
│  │ 阅读专题 →  │  │ 阅读专题 →  │       │
│  └───────────┘  └───────────┘       │
├─────────────────────────────────────┤
│  「更多专题即将上线」占位卡片          │
│  关联链接：工具中心 / 场景包 / 指南     │
└─────────────────────────────────────┘
```

**数据流**：
1. 页面加载时调用 `prisma.topic.findMany({ where: { status: "published" } })`
2. 按 `publishedAt` 降序排列
3. 每张卡片展示：封面 Emoji、标题、副标题、摘要、标签、适合人群、条目数、视频标记
4. 点击卡片 → 跳转至 `/topics/{slug}` 详情页

### /topics/{slug} — 专题详情页

展示完整的专题内容：
- Hero 区（标题 + 摘要 + 评级徽章）
- APP/工具条目列表（评级卡片 + 详细信息）
- 补充章节（引言、技巧、警告等）
- YouTube 视频嵌入（如果配置了）

### /resources — 资源库页

按分类（life/logistics/business/templates）分组展示所有已启用的资源链接。

### /guides — 指南页

展示所有已发布的文章，按类别分组。

## 五、常见问题

**Q：专题创建后 Slug 能改吗？**
A：不能。Slug 是数据库唯一键，创建后固定。如需修改，需新建专题并删除旧的。

**Q：如何隐藏一个已发布的专题？**
A：在后台编辑该专题，将状态改为 `archived`（归档）或 `draft`（草稿），前端将不再显示。

**Q：专题中的 YouTube 视频怎么添加？**
A：在 Basic 标签页粘贴完整的 YouTube URL（如 `https://www.youtube.com/watch?v=dQw4w9WgXcQ`），系统自动解析视频 ID、标题和封面图。

**Q：资源和文章有什么区别？**
A：资源 = 外部链接（如物流公司官网）；文章 = 站内长文内容（如集运攻略）。资源更像导航书签，文章是阅读内容。

**Q：专题的评级（S/A/B/C/D）怎么用？**
A：评级用于专题条目中，帮助用户快速判断 APP/工具的推荐程度。S 级为强烈推荐，D 级为不推荐。

## 六、快速操作清单

| 操作 | 入口 URL | 关键步骤 |
|------|---------|----------|
| 新建专题 | `/admin/topics` → 新建 | 填 Slug → 填基本信息 → 添加条目 → 状态改 published |
| 编辑专题 | `/admin/topics` → 点击专题 | 修改基本信息 / 条目 / 章节 → 保存 |
| 新增资源 | `/admin/resources` → 新增 | 填名称/URL/分类 → 勾选启用 → 保存 |
| 新增文章 | `/admin/cms` → 新建 | 填标题/Slug/正文 → 状态改 published → 保存 |
| 查看前端效果 | `/topics` `/resources` `/guides` | 直接访问对应页面 |
