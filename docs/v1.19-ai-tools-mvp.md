# v1.19 站内轻量 AI 工具 MVP — 开发记录

## 版本信息

- **前置版本**: v1.18.1-stable
- **当前版本**: v1.19-stable
- **定位**: 站内 AI 工具 MVP，让平台从"导航+工具站"升级为"站内能直接完成任务的 AI 工具站"

## 目标

1. 3 个 AI 工具：商品文案、翻译润色、合同摘要
2. 权限/额度系统（guest 1次, user 3次, member 50次, admin 999次）
3. 积分扣减（user 超免费次数后每次 20 分）
4. Mock 模式（无 API key 时可测试）
5. 安全降级（AI 未配置返回 503，不影响全站）

## 数据库

### 新增模型：AIUsageLog

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | cuid |
| userId | String? | 关联用户（可为 null = guest） |
| toolType | String | product_copy / translate_polish / document_summary |
| inputHash | String? | 输入 SHA256 哈希（不存原文） |
| outputTokens | Int? | 输出 token 数 |
| costPoints | Int | 消耗积分（0 或 20） |
| ipHash | String? | IP 哈希（guest 用） |
| userAgent | String? | 浏览器标识 |
| createdAt | DateTime | 创建时间 |

Migration: `20260517000000_add_ai_usage_log`

## 部署注意

### ⚠️ 关键修复：.env.production DATABASE_URL

v1.19 部署时发现 `.env.production` 中的 DATABASE_URL 指向本地 PostgreSQL (`127.0.0.1:5432/bxb_prod`)，而非 Neon 数据库。Migration 是在 Neon 上执行的，但运行时连接到本地库导致找不到 `ai_usage_logs` 表。

**修复**: 更新 `.env.production` 的 DATABASE_URL 为 Neon 池化 URL。

**教训**: 每次添加新 migration 后，必须确认 `.env.production` 的 DATABASE_URL 与迁移目标一致。
