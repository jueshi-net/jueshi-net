## VPS 关键修复记录

### 1. DATABASE_URL 密码编码问题

**问题**：VPS 生产数据库密码包含 `!` 和 `@` 特殊字符，在 DATABASE_URL 中必须 URL 编码。

- `!` → `%21`
- `@` → `%40`

**正确格式**：
```
DATABASE_URL=postgresql://bxb_user:Bxb2024%21Prod%40Secure@127.0.0.1:5432/bxb_prod?schema=public
```

**错误示例**（会导致 Prisma 认证失败）：
```
DATABASE_URL=postgresql://bxb_user:Bxb2024!Prod@Secure@127.0.0.1:5432/bxb_prod?schema=public
```

注意：`@` 字符同时出现在密码中和服务地址分隔符中，不编码会导致 Prisma 解析错误。

### 2. PM2 环境变量固化问题

**问题**：PM2 通过 `pm2 save` 保存环境变量快照。修改 `.env` 后直接 `pm2 restart` 不会加载新变量。

**正确操作顺序**：
```bash
# 1. 修改 .env 或 .env.production
# 2. 重启时强制加载环境变量
pm2 restart xixiong-saas --update-env
# 3. 验证进程环境变量
cat /proc/$(pgrep -f "next start" | head -1)/environ | tr '\0' '\n' | grep DATABASE
# 4. 确认正确后再 save
pm2 save
```

**排错方法**：
```bash
# 查看 PM2 实际加载的 DATABASE_URL
pm2 env 0 | grep DATABASE_URL

# 查看进程实际环境变量
cat /proc/$(pgrep -f "next start" | head -1)/environ | tr '\0' '\n' | grep DATABASE
```

### 3. AI 环境变量确认

**问题**：VPS `.env.production` 缺少 AI 相关变量，导致 AI API 返回 503。

**必须在 VPS .env.production 中确认**：
```
AI_ENABLED=true
AI_MOCK=true
AI_API_BASE_URL=
AI_API_KEY=
AI_MODEL=gpt-4o-mini
```

注意：`AI_API_KEY` 留空时，`AI_MOCK=true` 会启用 Mock 模式，返回模拟内容而不调用真实 API。

## 安全修复

### 文章 API 草稿泄露

| API | 修复前 | 修复后 |
|-----|--------|--------|
| GET /api/articles | 返回所有文章（含 draft） | 非 admin 只返回 published |
| GET /api/articles/[slug] | 返回任何状态的文章 | 非 admin 访问非 published → 404 |
| /admin/cms 后台 | 依赖同一个 API | 检测 admin session，返回全部 |

### 资源库 API

- GET /api/resources：非 admin 只返回 `isActive=true` 的资源
- **草稿状态说明**：`resources` 表只有 `isActive` 布尔字段，没有 draft/published 状态机。当前全部资源均视为公开资源（`isActive=true`）或隐藏资源（`isActive=false`），不存在 draft 泄露问题。
