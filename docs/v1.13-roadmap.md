## 二、v1.13 优先级规划

### P0 — 安全与稳定性（必须做）

| 优先级 | 项目 | 说明 | 预计工时 |
|--------|------|------|----------|
| P0-1 | **异地数据库备份** | 当前仅有本地 pg_dump，无异地备份。设计 R2/B2/另一台 VPS 方案 | 1-2 天 |
| P0-2 | **会员权限服务端校验** | 当前部分功能仅靠 localStorage.bxb_role 判断，需服务端 API 层统一校验 | 2-3 天 |
| P0-3 | **PNG 导出服务端兜底** | iPhone Safari html2canvas 兼容问题，设计 Playwright 服务端导出方案 | 2-3 天 |

### P1 — 社区与会员（重要但可延后）

| 优先级 | 项目 | 说明 | 预计工时 |
|--------|------|------|----------|
| P1-1 | **Discourse 社区 SSO** | 论坛独立部署，主站 SSO 登录，角色映射 | 3-5 天 |
| P1-2 | **User 模型论坛预留字段** | forumUserId、forumUsername 等字段设计 | 0.5 天 |

## 三、实施顺序建议

```
Phase 1: P0-2 会员权限服务端校验  ← 必须先做（安全基础）
Phase 2: P0-1 异地数据库备份      ← 数据安全保障
Phase 3: P0-3 PNG 导出服务端兜底  ← 用户体验修复
Phase 4: P1-1~P1-4 社区论坛       ← 新增功能入口
Phase 5: P2 内容与广告             ← 增长优化
```

---

## 四、风险提醒

1. **会员权限服务端校验** 必须在社区论坛之前完成，否则论坛 SSO 无法正确映射角色
2. **异地备份** 必须在社区论坛上线前就绪，避免多数据库后备份复杂度增加
3. **社区论坛** 依赖主站会员系统稳定，不建议在 Stripe 订阅逻辑仍在变动时上线
4. **VPS 资源**：8GB KVM 可支撑当前主站 + Discourse，但需监控内存使用
