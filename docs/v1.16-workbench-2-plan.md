## 一、功能目标

### 1. 自定义网址收藏

用户在作台添加自己的常用网址。

**字段设计 (WorkbenchLink 表)**:
- `id` — String @id @default(cuid())
- `userId` — String
- `title` — String (max 80)
- `url` — String (http/https)
- `description` — String? (max 200)
- `iconUrl` — String?
- `category` — String? (可选分类)
- `sortOrder` — Int @default(0)
- `createdAt` — DateTime @default(now())
- `updatedAt` — DateTime @updatedAt

**权限与限制**:
- 只能看自己的链接 (WHERE userId = session.userId)
- 登录用户才能添加/编辑/删除
- title ≤ 80 字, url 必须 http/https, description ≤ 200 字
- URL 安全校验：禁止 javascript:, data:, file: 协议
- 免费用户 (user): 最多 20 个
- Member/Admin: 最多 200 个
- 删除：直接硬删除（简化实现）

### 2. 站内工具收藏

用户收藏站内工具，dashboard 显示"我的常用工具"。

**字段设计 (ToolFavorite 表)**:
- `id` — String @id @default(cuid())
- `userId` — String
- `toolKey` — String (如 "documents", "label-maker", "postal-code" 等)
- `sortOrder` — Int @default(0)
- `createdAt` — DateTime @default(now())
- `@@unique([userId, toolKey])` — 同一用户同一工具只能收藏一次

**规则**:
- 一个用户同一个 toolKey 只能收藏一次
- 可以取消收藏 (DELETE)
- 用户没有收藏时，显示系统默认推荐工具
- 默认推荐工具列表定义在 src/lib/data/ 中

### 3. 身份选择 / 场景推荐

用户选择自己的身份，用于工作台推荐。

**实现**: User 表增加 `profileType` 字段

**可选值**:
- `merchant` — 出海商家
- `overseas_chinese` — 海外华人
- `student` — 留学生
- `ai_learner` — AI 学习者
- `logistics` — 物流/集运用户
- `none` — 暂不选择 (默认/空值)

**推荐包**:

| 身份 | 推荐工具 |
|------|---------|
| 出海商家 | 外贸单据、HS 编码查询、运费计算、AI 商品文案(Coming Soon)、出海做生意 |
| 海外华人 | 邮编查询、物流查询、反向海淘(Coming Soon)、Memo、自定义网址 |
| 留学生 | AI 学习、文献/论文工具(Coming Soon)、翻译润色(Coming Soon)、Memo、自定义网址 |
| AI 学习者 | AI 学习、AI 工具导航、提示词收藏(Coming Soon)、自定义网址 |

### 4. 工作台页面改版

**分区设计**:

```
┌─────────────────────────────────────────┐
│  顶部：积分、今日获得、签到、会员状态      │
│  [签到按钮]                              │

│  我的常用工具 (收藏工具 / 默认推荐)         │
│  我的自定义网址 (收藏链接)                 │
│  今日任务                                 │
│  场景推荐包 (根据 profileType)            │
├─────────────────────────────────────────┤
│  积分兑换入口                             │
│  积分流水                                 │
│  最近积分记录                             │
│  常用系统工具                             │

```

**移动端要求**:
- 卡片布局，不要横向超宽
- 按钮明显、可点击
- iPhone Safari 可用

## 二、API 列表

### 1. GET /api/workbench/summary
**认证**: requireLogin()
**返回**:
```json
{
  "profileType": "merchant" | null,
  "links": [{ id, title, url, description, iconUrl, category, sortOrder }],
  "favorites": [{ toolKey, sortOrder }],
  "recommendedTools": [{ key, title, href, icon }],
  "recommendedPackages": [{ name, tools: [{ key, title, href }] }],
  "limits": { maxLinks: 20, maxFavorites: 999 },
  "points": 150,
  "memberInfo": { isMember: false, memberUntil: null }
}
```

### 2. POST /api/workbench/links
**认证**: requireLogin()
**Body**: `{ title, url, description?, iconUrl?, category? }`
**校验**:
- title 非空且 ≤ 80
- url 合法 http/https
- description ≤ 200 (可选)
- 数量限制检查
**返回**: 201 { link }
**错误**: 401/400/403/409

### 3. PATCH /api/workbench/links/[id]
**认证**: requireLogin()
**Body**: 同 POST
**校验**: userId 匹配
**返回**: 200 { link }
**错误**: 401/403/404/400

### 4. DELETE /api/workbench/links/[id]
**认证**: requireLogin()
**校验**: userId 匹配
**返回**: 204
**错误**: 401/403/404

### 5. POST /api/workbench/favorites
**认证**: requireLogin()
**Body**: `{ toolKey }`
**校验**: toolKey 非空，不重复
**返回**: 201 { favorite }
**错误**: 401/400/409

### 6. DELETE /api/workbench/favorites/[toolKey]
**认证**: requireLogin()
**校验**: userId 匹配
**返回**: 204
**错误**: 401/403/404

### 7. PATCH /api/workbench/profile
**认证**: requireLogin()
**Body**: `{ profileType }`
**校验**: profileType 在允许值列表中
**返回**: 200 { profileType }
**错误**: 401/400

### 8. PATCH /api/workbench/reorder (v1.16 可选)
**说明**: 第一版如复杂可先不做，预留 API 路径。

## 三、数据库 Migration

### 新增表

**WorkbenchLink**
```
model WorkbenchLink {
  id          String   @id @default(cuid())
  userId      String
  title       String   // max 80
  url         String   // http/https
  description String?  // max 200
  iconUrl     String?
  category    String?
  sortOrder   Int      @default(0)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId])
  @@map("workbench_links")
}
```

**ToolFavorite**
```
model ToolFavorite {
  id        String   @id @default(cuid())
  userId    String
  toolKey   String
  sortOrder Int      @default(0)
  createdAt DateTime @default(now())

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([userId, toolKey])
  @@index([userId])
  @@map("tool_favorites")
}
```

### User 表变更

```
// 新增字段
profileType String? @map("profile_type")
```

### Migration 命名

`20260516_add_workbench_tables`

### 注意事项

- 使用 `npx prisma migrate dev --name add_workbench_tables` 本地生成
- 生产用 `npx prisma migrate deploy`
- `@@map` 使用 snake_case 表名，字段用 camelCase
- 关联使用 onDelete: Cascade 确保用户删除时清理数据

### 数据库回滚
```bash
# 如果 migration 有问题
DATABASE_URL="..." npx prisma migrate resolve --rolled-back 20260516_add_workbench_tables
# 或手动 DROP TABLE
# 然后从备份恢复：
gunzip -c /var/backups/xixiong-saas/postgres/bxb_prod_20260516_014543.sql.gz | \
  PGPASSWORD="..." psql -h 127.0.0.1 -U postgres -d bxb_prod
```

### 回滚决策点
- 如果 build 失败 → 不回滚 DB，只回滚代码
- 如果 migration 失败但未执行完 → 手动清理后回滚
- 如果页面 500 → 立即回滚代码，DB 保留（新表不影响旧功能）
